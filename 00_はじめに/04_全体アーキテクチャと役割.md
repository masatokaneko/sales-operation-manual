## 0. 全体アーキテクチャ

| 層                                    | 主体                        | 主目的                   | 担当領域                                     |
| ------------------------------------ | ------------------------- | --------------------- | ---------------------------------------- |
| **Front Office（営業推進層）**              | AE／SA                     | 提案活動（顧客接点・課題抽出・提案・交渉） | 顧客訪問、提案書、見積依頼、議事録、Forecast               |
| **Middle Office（営業管理層・Corporate所管）** | Sales Operation／Deal Desk | 商談管理・審査・収益統制          | Salesforce運用、見積承認、Deal Desk、WPS推進、契約手続管理 |
| **Back Office（経理・法務層・Corporate所管）**  | Finance／Legal             | 契約締結・請求・収益認識          | NDA／契約書／請求書／発注書管理、会計処理、インセンティブ処理         |

### 0.1. SAチームの役割分担

本マニュアルにおけるSA（Solution Architect）は、以下の3つの専門チームで構成される。

- **Discovery SA:** 顧客と対面し、課題発掘・技術検証を担当。顧客のエンジニアと対話し、技術的な現状（As-Is）と課題を深く理解する。
- **Demo SA:** 製品を起点にした環境構築やアプリケーション開発、顧客要望によるカスタマイズを行い、Discovery SAの提案活動をサポート。
- **Post-Sales SA:** 顧客プロジェクトに伴走し、ScalarDBを用いた環境構築やアプリケーション開発のサポートを担当。

> [!NOTE]
> SAの詳細な役割定義、主要ミッション、行動原則については、[00_はじめに/03_ScalarのAE・SAのあるべき姿と提案の定義.md](./03_ScalarのAE・SAのあるべき姿と提案の定義.md)を参照してください。

各プロセスにおけるSAの関与は、この役割分担に基づき行われる。

## Ⅱ. Front / Middle / Back の分担責任（RACI形式）

| 業務領域              | Front（AE／SA） | Middle（SalesOps／DealDesk） | Back（Finance／Legal） |
| ----------------- | ------------ | ------------------------- | ------------------- |
| 顧客訪問・提案           | R            | C                         | I                   |
| 提案書／見積依頼          | R            | A                         | I                   |
| Salesforce登録・更新   | R            | A                         | I                   |
| Deal Desk審査       | C            | R/A                       | I                   |
| WPS（Win Planning） | R/A          | C                         | I                   |
| 契約書レビュー           | I            | C                         | R/A                 |
| 見積／発注書整合チェック      | C            | R                         | A                   |
| 請求書発行／収益認識        | I            | C                         | R/A                 |
| スプリット（インセンティブ申請）  | R            | A                         | C                   |

## Ⅳ. ガバナンス構造図（推奨運用）

```
[Front Office] AE/SA
 ├─ 日次：朝会・顧客訪問・議事録
 ├─ 週次：Forecast／WPS参加
 └─ 四半期：アカウントプラン更新
        │
        ▼
[Middle Office] SalesOps (Corporate管轄)
 ├─ Salesforce統制／見積管理
 ├─ Deal Desk（審査・承認）
 └─ WPS支援（提案推進・資料レビュー）
        │
        ▼
[Back Office] Finance／Legal
 ├─ 契約管理／Docusign運用
 ├─ 請求・収益認識・会計仕訳
 └─ インセンティブ／スプリット処理
```
