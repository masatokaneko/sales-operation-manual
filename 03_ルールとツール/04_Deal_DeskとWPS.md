## 2️⃣ 商談統制／推進（Middle Office：Corporate所管）

Middle Officeは**「牽制（Deal Desk）」と「推進（WPS）」**を分離して運営する。

---

### 2.1 Deal Desk（牽制・審査会）

**目的**：案件のリスク・価格・契約統制・コンプライアンス審査
**開催**：毎週金曜午前（COO／CFO主催）
**出席**：SalesOps・Legal・Finance・AE・SA（必要時）

| 審査項目 | 内容                 |
| ---- | ------------------ |
| 商流   | 直販／代理店（商流経路・再販有無）  |
| 契約先  | T&Cs締結先・開発委託先      |
| 値引   | 条件・理由・利益率・承認可否     |
| 契約形態 | NDA／T&Cs／業務委託／取次形態 |
| リスク  | 契約リスク／納期リスク／法的リスク  |
| 収益性  | 原価率・マージン・支払サイト     |
| 承認結果 | 承認／差戻し／再審査日程       |

**ルール**

* Quip「Deal Desk Log」にAEが前日までに登録。
* Slackや口頭での承認は無効。Quip承認記録が唯一の証憑。
* Deal Desk承認なし案件は収益認識不可。

---

### 2.2 WPS（Win Planning Session：推進会議）

**目的**：案件獲得に向けた提案戦略の議論・支援（Deal Deskとは別）
**開催**：週次（木曜午後）
**参加者**：AE・SA・Mgr・SalesOps
**内容**：

1. 商談の背景・課題・キーマン状況
2. 提案ストーリー・競合差別化要素
3. 成約阻害要因と打ち手
4. 社内サポート・リソース配分検討

**ルール**

* Deal Deskで扱う「リスク案件」以外を対象。
* AE主導で事前資料（Discovery Map＋提案骨子）をQuipに登録。
* 決定事項は「Next Action」としてSlackに転送。
