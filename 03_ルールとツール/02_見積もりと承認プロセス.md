### 1.4 見積書ルール

| 項目   | 内容                                                                                       |
| ---- | ---------------------------------------------------------------------------------------- |
| 見積種別 | **概算見積**（提案用・予算取り目的・契約義務なし）／**正式見積**（契約用）                                                                    |
| ツール  | Freeeで作成（AEが作成責任を持つ）                                                                     |
| 承認フロー  | **通常（値引きなし）**：<br>- AEがFreee下書き作成<br>- SalesOpsが条件チェック<br>- 営業マネジャー承認のみで完了<br><br>**値引き・特殊条件あり**：<br>- AEがFreee下書き作成<br>- SalesOpsが通常条件との差分をチェック<br>- 営業マネジャー承認<br>- CxO最終承認（必須） |
| ルール  | - 正式見積書には見積No、対象製品、期間、金額を明記<br>- 顧客提出前にQuipでレビュー記録必須<br>- 承認済み見積は編集不可 |
| 値引規定 | 原則禁止。例外：<br>①12か月超契約<br>②複数年コミット<br>③事例化前提（CxO承認必須）                                  |
| 価格基準 | Scalar価格表最新版（更新日を見積書内に記載）                                                                |

### 社内承認フロー

| フェーズ     | 承認ツール | 承認者                        | 備考       |
| -------- | ----- | -------------------------- | -------- |
| 提案書／概算見積 | Quip  | Deal Desk（TAD, Wataru, 金子） | AEが申請    |
| 正式見積     | Freee | 深津・金子                      | CxO承認必須  |
| 請求書      | Freee | 佐藤・金子                      | コーポレート発行 |
| 納品書      | Freee | 深津・金子                      | 営業オペ発行   |

### Deal Desk運用

- **開催頻度**：毎週金曜日
- **臨時開催**：Sales Operation経由で依頼
- **審査対象**：商流、契約先、値引、契約形態、リスク、収益性の審査・承認
- **議事録**：Confluenceに記載
- **注意事項**：Deal Deskは牽制・審査会であり、提案推進はWPS（Win Planning Session）で実施
