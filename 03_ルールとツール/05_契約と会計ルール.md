## 3️⃣ 契約・会計処理（Back Office：Corporate所管）

---

### 3.1 契約種別と定義

| 種類           | 概要                                     | 備考                              |
| ------------ | -------------------------------------- | ------------------------------- |
| **ライセンス契約**  | Scalar製品の使用権を許諾する契約。利用条件・期間・金額は発注書で定義。 | 「Terms & Conditions（T&C）」が基礎契約。 |
| **業務委託契約**   | コンサル・開発・保守等を委託する契約。請負または準委任。           | Scalarは主に準委任契約を採用。              |
| **特別サポート契約** | 通常サポートより手厚い対応を提供する追加契約。                | Add-On Supportとして見積明細に記載。       |

### 3.2 契約手続フロー（詳細プロセス）

| フェーズ    | 書類                      | 発行／受領                        | 承認ルール                          |
| ------- | ----------------------- | ---------------------------- | ------------------------------ |
| ① 情報共有  | **NDA**                 | AE依頼 → Legal発行／締結            | 顧客との初回情報交換前に必須                 |
| ② 提案／見積 | **見積書**                 | AE作成（Freee） → SalesOps／CxO承認 | 価格表遵守・Quip記録必須                 |
| ③ 契約締結  | **T&Cs／業務委託契約書**                | Corporate発行／**Docusign**（原則電子契約）       | **CEO／CFO承認必須**・法務レビュー経由                |
| ④ 発注    | **発注書**                 | 顧客発行 → AE受領 → Finance登録      | 見積No必須。誤り時は再発行依頼               |
| ⑤ 納品・検収 | **納品書・検収書**（業務委託のみ） | 営業オペ作成・Freee押印申請 | ライセンス契約は利用開始通知のみ |
| ⑥ 売上報告  | **Salesforce更新／Quip報告** | AE／SalesOps                  | Stage6時点で確定登録                  |
| ⑦ 請求    | **請求書**                 | Finance発行（Freee）             | 発注書受領後のみ発行可（原則：発注書単位で1対1）                    |
| ⑧ 収益認識  | **RevRec登録**            | Finance                      | 発注書受領時に契約資産／前受収益計上→期間按分で売上計上                |
| ⑨ スプリット | **インセンティブ申請**           | AE → SalesOps承認 → CFO確定      | Quip「Split申請フォーム」使用。収益確定後のみ提出可 |

---

### 3.2 証憑チェックリスト（中間検査）

| 種別      | 必要書類                         | 管理責任             | 確認内容         |
| ------- | ---------------------------- | ---------------- | ------------ |
| ライセンス直販 | NDA／T&Cs／見積書／発注書／請求書         | SalesOps／Finance | 顧客・金額・期間の整合性 |
| 代理店販売   | NDA／T&Cs（3者）／発注書（代理店→Scalar） | SalesOps         | 商流・再販条件確認    |
| 業務委託    | 業務委託契約／検収書／請求書               | Legal／Finance    | 納品物の有無と受領証跡  |
| 取次商品    | NDA／取次契約書／発注書                | SalesOps         | 収益区分の明確化     |

---

### 3.4 会計・収益認識ルール

* **契約＋発注＋請求書**の3点完備をもってRevRec可能。
* 発注書に見積Noがない場合、収益認識不可。
* 契約締結日＝収益開始日ではなく、**利用開始日ベースで按分計上（日割／月割）**。
* Freeeでの仕訳はFinanceが実施。AE・SalesOpsは計上内容を閲覧可能。
* 売上報告は月次でCFOがCEOに報告。

### 3.5 主なリスクと対策

| リスク          | 対応策                   |
| ------------ | --------------------- |
| 発注書未受領のまま計上  | 発注書No.を確認、未受領案件は計上禁止  |
| 契約期間誤認による誤按分 | 契約期間をFreee・SFDCで一致管理  |
| 入金遅延         | 入金確認・リマインドフローを定期化     |
| 見積と承認内容不整合   | Quip／Freee間の添付整合性を義務化 |
